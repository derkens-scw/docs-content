This section contains usage limits that apply when using Messaging and Queuing.


| Resource                                       | Criteria   | Limits   | Scope        |
|------------------------------------------------|------------|----------|--------------|
| NATS Account                                   | Max number | 10       | Organization |
| NATS message payload                           | Max size   | 512 KB   | Message      |
| NATS queue/stream storage                      | Max size   | 300 MB** | NATS Account |
| Max number of NATS streams                     | Max number | 10       | NATS Account |
| Max number of NATS consumers                   | Max number | 50       | NATS Account |
| Max number of SNS subscribers per Topic        | Max number | 50       | Project      |
| SQS/SNS message payload                        | Max size   | 256 KB   | Message      |
| Max number of SQS Queues                       | Max number | 15       | Project      |
| SQS queue storage                              | Max size   | 100 MB   | Project      |
| Message Retention Period (SQS)                 | Max number | 14 days  | Message      |
| Visibility Time Out (SQS)                      | Max number | 12 hours | SQS Queue    |
| Max number of SNS topics                       | Max size   | 15       | Project      |

**This implies stream replication. If you choose to replicate your stream across all nodes (3), a storage limit of 100 MB applies.

These limits are enforced as [Organization quotas](/console/my-account/reference-content/organization-quotas/).
